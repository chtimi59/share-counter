It's commonly admitted, that the "Five Ws model" are questions whose answers are considered basic in information-gathering.

Five "W" for:

  • WHO did that?
  • WHAT happened?
  • WHEN did it take place?
  • WHERE did it take place?
  • Why did it take place?

It's often used in journalism, software development or in research, in order to make a full description of an issue.

If we now forget the lastest "Why", we get here a really minimal setup for a Cloud Databases which actually fits with very a large application fields.

Car's Parking Inventory

  • WHO: The identity of the guy (or system) who share the info.
  • WHAT: Number of remaining place
  • WHEN: When the observation was made
  • WHERE: GPS position of the parking's place

Lab's Temperature measurement

  • WHO: The identity of the system which made the measurement.
  • WHAT: Temperature
  • WHEN: When the measurement was made
  • WHERE: GPS position of the sensor

Post'it sharing

  • WHO: The author of the Post'it
  • WHAT: A little text
  • WHEN: When he writes it

Line up management

  • WHAT: Current ticket Number
  • WHEN: When was it updated

So based on this, Share-Counter has been developed to save time and money to avoid re-inventing the wheel everytime.

How does it works?

Once you've create an account, you can create some counters.

A counter is bascially a database with the 4W fields, which are optionals (depending of your needs)

  • WHO = ['UserName']
  • WHAT = ['Text', 'Value'] (depending if you want to store numerical value or not
  • WHEN = ['Date'] (UTC date)
  • WHERE = ['Location'] (Json latitude/longitude)
